## 工作区标签系统

### 标签继承机制

工作区支持两种标签：

1. **继承标签**：自动从包含的项目收集
   - 实时计算，不持久化
   - 自动去重
   - 可选择关闭继承

2. **自定义标签**：用户手动添加
   - 持久化存储
   - 可随时增删
   - 优先级高于继承标签

### 使用场景

```
场景：管理微服务项目组

工作区：订单服务组
├─ 订单API项目 [标签: backend, api, dotnet]
├─ 订单Web项目 [标签: frontend, web, react]
└─ 订单数据库 [标签: database, sql]

自动继承标签：[backend, api, dotnet, frontend, web, react, database, sql]
用户自定义标签：[microservice, order-service, production]

最终标签：[microservice, order-service, production, backend, api, dotnet, 
          frontend, web, react, database, sql]
```

### 标签筛选

点击标签可以筛选：
- **项目标签**：显示包含该标签的所有项目
- **工作区标签**：显示包含该标签的所有工作区
- **混合视图**：同时显示匹配的项目和工作区

## IDE 管理

### 支持的IDE

| IDE | 标识 | 自动检测路径 |
|-----|------|-------------|
| Visual Studio Code | vscode | `%PATH%\code.exe` |
| Trae | trae | `%LocalAppData%\Trae\Trae.exe` |
| Trae-CN | trae-cn | `%LocalAppData%\Trae CN\Trae.exe` |
| 自定义 | custom | 用户手动选择 |

### IDE检测策略

1. **环境变量检测**
   - 检查PATH环境变量
   - 查找可执行文件

2. **常见路径扫描**
   - Windows: `%LocalAppData%`, `%ProgramFiles%`
   - macOS: `/Applications`, `~/Applications`
   - Linux: `/usr/bin`, `/usr/local/bin`

3. **注册表检测（Windows）**
   - 查询安装信息
   - 获取安装路径

4. **用户手动配置**
   - 文件选择对话框
   - 验证可执行文件
