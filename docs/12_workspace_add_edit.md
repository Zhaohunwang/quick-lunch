## 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 工作区名称 | 文本 | 是 | 工作区的显示名称 |
| 描述 | 文本 | 否 | 工作区的描述信息 |
| 自动继承项目标签 | 布尔 | 否 | 是否自动继承包含项目的标签（默认开启） |
| 自定义标签 | 标签列表 | 否 | 用户手动添加的标签 |
| 继承标签 | 标签列表 | 只读 | 自动从项目继承的标签（灰色显示） |
| 项目选择 | 多选 | 是 | 至少选择一个项目 |

## 标签管理功能

### 自动继承标签
- 当勾选「自动继承项目标签」时，系统会自动收集所有包含项目的标签
- 继承标签实时计算，不持久化存储
- 继承标签显示为灰色，不可直接删除

### 自定义标签
- 点击 [+] 按钮添加新标签
- 点击标签上的 [×] 删除自定义标签
- 自定义标签持久化存储
- 与继承标签去重合并

### 标签显示规则
```
工作区所有标签 = 自定义标签 ∪ 继承标签（去重）
```

## 交互流程

### 1. 打开对话框
- 点击工作区管理页面的「添加工作区」按钮
- 或点击现有工作区的「编辑」按钮

### 2. 输入基本信息
- 输入工作区名称
- 可选输入描述

### 3. 配置标签
- 勾选/取消「自动继承项目标签」
- 添加自定义标签
- 查看继承标签预览

### 4. 选择项目
- 从可选项目列表中勾选
- 使用搜索框快速查找
- 支持按标签筛选
- 拖拽调整项目顺序

### 5. 管理已选项目
- 点击×移除项目
- 拖拽调整顺序

### 6. 保存
- 验证至少选择一个项目
- 保存到数据库
- 刷新工作区列表
- 更新标签云

## 标签继承示例

### 场景：创建"电商平台"工作区

**包含项目：**
- 前端项目 [标签: web, frontend, react]
- 后端API [标签: backend, api, dotnet]
- 数据库 [标签: database, sql]

**自动继承标签：**
```
[web, frontend, react, backend, api, dotnet, database, sql]
```

**用户添加自定义标签：**
```
[e-commerce, production]
```

**最终工作区标签：**
```
[e-commerce, production, web, frontend, react, backend, api, dotnet, database, sql]
```

## 项目选择功能

### 搜索项目
- 实时搜索项目名称、别名
- 支持模糊匹配

### 快速选择
- **最近打开**：显示最近7天打开的项目
- **按标签筛选**：按标签分类显示项目
- **全部项目**：显示所有可用项目

### 已选项目管理
- 显示已选项目数量
- 显示项目标签预览
